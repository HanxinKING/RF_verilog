## 8. 重要注意事项：旧新混合运行产物

目录 `fpga_mix12_batch_7x50` 在调试阶段曾被多个 ModelSim 进程写入过，因此：

- `rtl_results.csv` 可能混有旧格式和新格式行
- 有些行是新的 5 列格式
- 更老的行仍可能是旧的 7 列格式

新的干净格式如下：

```csv
sample_idx,rtl_class,rtl_score,float_class,class_match
```

旧的混合格式如下：

```csv
sample_idx,rtl_class,rtl_score,expect_class,expect_score,class_match,score_match
```

因此，不能只根据 `rtl_results.csv` 的总行数判断结果是否可靠。

如果需要使用干净快照，应优先使用或重新生成：

- `F:\RF_cnn\CONV1_21\project_1\mem\fpga_mix12_batch_7x50\rtl_results_float_only_clean.csv`

## 9. 交接时的运行状态

在交接文档编写时，系统中仍有多个 `vsim / vsimk` 进程残留。

这意味着新的会话在继续正式批量运行前，必须先检查并停止旧的 ModelSim 进程，否则很容易污染批量目录。

混合运行的典型症状包括：

- `rtl_results.csv` 行数大于 350
- 最后一行是半写入或格式异常
- `modelsim_stdout.log` 中的进度与 csv 行数不匹配

## 12. 关键结果文件说明

### `rtl_results.csv`

按样本记录最终分类结果。

当前预期格式为：

- `sample_idx`
- `rtl_class`
- `rtl_score`
- `float_class`
- `class_match`

### `rtl_logits.csv`

按样本、按类别记录 RTL 原始分数。

字段：

- `sample_idx`
- `class_idx`
- `score`

这个文件适合用来分析 RTL 分数分布与浮点 logit 的对应关系。

### `summary_float_fixed.json`

仅包含 Python 浮点与 Python 定点的汇总结果。

包括：

- `float_accuracy`
- `fixed_accuracy`
- `float_fixed_agreement`
- 各层指标：
  - `mean_abs_err`
  - `max_abs_err`
  - `cosine_similarity`

### `summary_final.json`

浮点、定点、RTL 三者的综合汇总。

只有在干净运行目录下生成时，这个文件才可信。
