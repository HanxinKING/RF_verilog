# 项目汇报 PPT 大纲

## 从哪里开始

建议你不要一上来就讲代码或模块，而是先把整件事讲成一条清楚的主线：

1. 这个项目要解决什么问题。
2. 为什么要把模型部署到 FPGA/RTL 链路。
3. 我们采用了什么新流程完成从浮点模型到 RTL 验证。
4. 最终验证结果说明了什么。

一句话版本可以直接作为第一页的核心表述：

`本项目目标是将 7 类 SlimSEI 信号识别模型部署到 FPGA 友好的 RTL 路径，并验证 Python 浮点、Python 定点和 ModelSim RTL 三者的一致性与性能表现。`

## 推荐页数

建议先做 10 页左右，结构最稳：

1. 项目背景
2. 项目目标
3. 整体技术路线
4. 模型与数据集
5. 工程实现流程
6. RTL 关键设计
7. 已解决的关键问题
8. 最终验证结果
9. 问题分析与改进方向
10. 总结

## 每页该讲什么

### 第 1 页 项目背景

建议标题：

`基于 FPGA 的 7 类 SlimSEI 信号识别部署与验证`

这一页讲：

- 原始模型来自 SlimSEI。
- 目标不是只在 Python 里跑通，而是要进入 FPGA/RTL 可部署路径。
- 工程难点在于浮点模型、定点量化、RTL 仿真三条链路必须统一。

可以放的材料：

- 一张总体框图：数据输入 -> 浮点模型 -> 定点量化 -> RTL 推理 -> 结果比对。

### 第 2 页 项目目标

这一页要明确讲清楚验收标准：

- 建立 `fpga_mix12` 新流程。
- 完成 Python 浮点参考输出。
- 完成 Python 定点参考输出。
- 完成 ModelSim RTL 批量仿真输出。
- 对比三者分类结果与逐层误差。

可以直接写成 3 级目标：

- 功能目标：链路跑通。
- 一致性目标：RTL 对齐 Python 定点。
- 性能目标：观察相对真实标签的分类效果。

### 第 3 页 整体技术路线

这是整套 PPT 的核心页，建议画流程图。

建议流程：

1. 从 Keras/HDF5 模型加载浮点权重。
2. 运行浮点推理，生成参考分类结果与中间层输出。
3. 做量化与校准，导出 `.mem` 参数文件。
4. 在 RTL 中加载权重和输入数据。
5. 使用 ModelSim 跑批量仿真。
6. 输出 `rtl_results.csv` 和 `rtl_logits.csv`。
7. 与浮点/定点结果做一致性分析。

建议重点强调：

- 当前生产路径是 `fpga_mix12`。
- 不是旧的 `CNN_RF` 批处理路径。

### 第 4 页 模型与数据集

这一页讲模型来源和验证集构成。

建议内容：

- 模型来源：`SparseComplexCNNSameChannel35.hdf5`
- 类别数：7 类
- 当前正式验证集：`7 x 50 = 350` 个样本
- 每类样本数均衡，便于观察每类准确率和混淆情况

这一页最好放：

- 模型文件路径
- 数据集路径
- 一个简化网络结构图

### 第 5 页 工程实现流程

这一页从“脚本和工程文件怎么配合”去讲。

可以分成两部分：

- Python/导出侧
- RTL/ModelSim 侧

建议放这些关键文件：

- `slimsei64_flow.py`
- `slimsei64_auto_calibrate.py`
- `export_fpga_mix12_batch.py`
- `cnn1d_complex_classifier_deploy_top_fpga_mix12.v`
- `cnn1d_dense_argmax_core_fpga_mix12.v`
- `cnn1d_complex_classifier_batch_fpga_mix12_tb.v`
- `run_fpga_mix12_batch.bat`

讲法建议：

- Python 侧负责浮点参考、定点量化和 `.mem` 导出。
- RTL 侧负责加载参数并完成时序推理。
- Testbench 负责批量输入、结果记录和进度输出。

### 第 6 页 RTL 关键设计

这一页别铺太多代码，讲“结构”和“数据流”。

建议讲：

- 复数卷积链路
- BN 和激活相关量化
- 多层卷积加池化级联
- `dense1 -> sr10 -> dense2 -> argmax`

最好强调：

- `dense2` 已经支持按类别独立 shift。
- 这是之前发现并修复过的关键问题之一。

### 第 7 页 已解决的关键问题

这一页非常适合展示你的工程价值。

建议列 3 个关键问题：

1. `.mem` 导出位宽错误问题
2. 单样本 dump testbench 循环变量干扰问题
3. `dense2` 每类别 shift 与 RTL 标量 shift 不一致问题

每个问题建议用同样结构描述：

- 问题现象
- 根因
- 修复方式
- 修复后的效果

这一页往往是答辩老师最愿意听的部分。

### 第 10 页 总结与下一步

总结建议写成 3 条：

- 已完成 `fpga_mix12` 新链路从浮点到 RTL 的完整打通。
- 已证明 RTL 与 Python 定点完全一致。
- 当前后续优化重点应放在量化策略和弱分类别提升，而不是 RTL 功能修错。

下一步建议：

1. 单独分析 `3/4/6` 类错分样本。
2. 对比浮点与定点在这些类别上的 logit 变化。
3. 优化量化参数或结构敏感层。
4. 如需上板，继续推进综合、时序和资源评估。

## 你做 PPT 时最好的起点

如果你现在马上开始做，我建议按这个顺序准备材料：

1. 先做第 1、2、3 页，把故事主线定下来。
2. 再做第 8、9 页，把最终结果页先固定。
3. 然后补第 5、6、7 页，把工程实现和关键问题填进去。
4. 最后补第 4 页和第 10 页，形成完整闭环。

这样做的好处是：

- 先把“为什么做”和“最后结果”讲清楚。
- 中间实现细节就不容易讲散。
- 你后面删页或压缩页数也更容易。
