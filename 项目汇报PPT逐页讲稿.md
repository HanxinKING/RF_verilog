## 第 1 页 封面页

### 页面标题

`基于 FPGA 的 7 类 SlimSEI 信号识别部署与验证`

### 页面要点

- 项目方向：深度学习模型 FPGA/RTL 部署验证
- 模型对象：7 类 SlimSEI 信号识别网络
- 验证链路：Python 浮点、Python 定点、ModelSim RTL
- 工程目标：建立可复现、可验证的 `fpga_mix12` 新流程

### 讲解话术

这一页我主要介绍本项目的总体内容。我们的目标是把 7 类 SlimSEI 信号识别模型，从原始的 Python 浮点实现，逐步推进到面向 FPGA 的 RTL 实现，并且对整个链路进行一致性验证。项目的核心不是只让模型在软件里跑通，而是要建立一条完整、稳定、可复现的 `fpga_mix12` 部署与验证流程。

## 第 2 页 项目背景

### 页面标题

`项目背景与研究意义`

### 页面要点

- 信号识别模型在软件侧易验证，但硬件部署难度更高
- FPGA 部署要求模型具备定点化、参数导出和时序实现能力
- 浮点模型、定点模型、RTL 结果必须逐步对齐
- 只有三者一致，才能说明硬件实现真正可信

### 讲解话术

这个项目的背景是，神经网络模型通常先在 Python 环境中训练和验证，但如果要真正部署到 FPGA，就必须解决定点量化、参数映射、RTL 实现和仿真验证的问题。也就是说，软件上的正确并不代表硬件上的正确。我们需要证明浮点模型、定点模型和 RTL 仿真输出之间具有稳定的一致性，这样后续上板实现才有可靠基础。

## 第 3 页 项目目标

### 页面标题

`项目目标`

### 页面要点

- 建立 `fpga_mix12` 新流程，替代旧批处理路径
- 生成 Python 浮点参考结果
- 生成 Python 定点参考结果
- 完成 ModelSim RTL 批量仿真
- 对比三者分类结果、混淆矩阵和逐层误差

### 讲解话术

本项目的目标可以分成三层。第一层是流程目标，也就是建立 `fpga_mix12` 这条新的正式验证路径。第二层是一致性目标，需要完成 Python 浮点、Python 定点和 RTL 三套结果的对比。第三层是结果分析目标，要在批量样本上统计准确率、混淆矩阵和每类表现，从而判断当前链路的整体质量以及后续优化重点。

## 第 4 页 整体技术路线

### 页面标题

`整体技术路线`

### 页面要点

- 从 Keras/HDF5 模型加载浮点权重
- 运行浮点推理，得到参考分类结果和中间层输出
- 执行量化与校准，导出 `.mem` 参数文件
- 在 RTL 中加载参数和输入样本
- 使用 ModelSim 完成批量仿真
- 输出 RTL 结果并与浮点、定点结果对比

### 讲解话术

整体流程是从浮点模型开始，先在 Python 中得到标准参考结果，然后做量化和校准，把权重、偏置以及相关参数导出为 RTL 可以直接读取的 `.mem` 文件。接着在 ModelSim 中运行 RTL 批量测试，把推理结果写出成结果表和 logits 表。最后，我们用这些结果去和 Python 浮点、Python 定点做系统对比，确认每一步的正确性。

## 第 5 页 模型与数据集

### 页面标题

`模型来源与验证数据`

### 页面要点

- 模型文件：`SparseComplexCNNSameChannel35.hdf5`
- 模型任务：7 类信号识别
- 正式验证集：`7 × 50 = 350` 个样本
- 每类样本数均衡，便于统计每类准确率
- 数据和模型均来自 SlimSEI 相关工程路径

### 讲解话术

本次验证使用的是 7 类 SlimSEI 模型，对应的模型文件是 `SparseComplexCNNSameChannel35.hdf5`。正式批量验证采用 350 个样本，也就是 7 类、每类 50 个样本的均衡数据集。这样的数据分布非常适合做每类准确率和混淆矩阵分析，因为不会被类别不平衡问题干扰。

## 第 6 页 工程实现流程

### 页面标题

`工程实现流程与关键文件`

### 页面要点

- Python 侧负责浮点参考、量化校准和 `.mem` 导出
- RTL 侧负责模型结构实现和推理数据通路
- Testbench 负责批量输入、结果记录和进度输出
- 关键脚本：`slimsei64_flow.py`、`export_fpga_mix12_batch.py`
- 关键 RTL：`deploy_top_fpga_mix12.v`、`dense_argmax_core_fpga_mix12.v`

### 讲解话术

从工程分工来看，Python 侧主要完成三件事：第一是生成浮点参考结果，第二是完成量化校准，第三是导出 RTL 可直接使用的参数文件。RTL 侧则负责实现网络的数据通路，包括卷积、池化、全连接和最终分类。Testbench 负责把批量样本送入 RTL，并在仿真过程中把分类结果和各类别分数输出成表格文件，供后续自动分析使用。

## 第 7 页 RTL 关键设计

### 页面标题

`RTL 关键结构设计`

### 页面要点

- 复数卷积与多层卷积池化级联
- BN 与激活相关量化处理
- `dense1 -> sr10 -> dense2 -> argmax` 输出链路
- 支持批量样本仿真与结果自动写出
- `dense2` 已支持按类别独立 shift

### 讲解话术

RTL 设计的重点在于如何把 Python 中的网络运算稳定映射到定点和时序逻辑上。当前链路包含多层卷积和池化，以及后端的 `dense1`、`sr10` 和 `dense2` 分类部分。这里一个很关键的改动是 `dense2` 现在已经支持按类别独立 shift，这解决了之前 Python 定点和 RTL 在最后分类阶段不一致的问题。

## 第 8 页 已解决的关键问题

### 页面标题

`项目中已解决的关键问题`

### 页面要点

- 修复 `.mem` 导出位宽错误问题
- 修复 dump testbench 循环变量干扰问题
- 修复 `dense2` 按类别 shift 与 RTL 标量 shift 不一致问题
- 清理批量运行中的旧新结果混写问题

### 讲解话术

这页主要体现工程推进中的关键工作。首先，之前旧路径存在 `.mem` 导出位宽错误的问题，会直接影响权重加载的正确性。其次，单样本 dump testbench 曾出现循环变量干扰，导致中间层导出异常。最后，也是最关键的一点，之前 Python 定点使用的是按类别的 `dense2_shift`，而 RTL 只用了一个统一标量，这会导致部分类别，尤其是 class 6 的结果偏差。现在这些问题都已经被定位并修复，整个链路已经稳定下来。

## 第 9 页 最终验证结果

`最终验证结果`

### 页面要点

- 浮点 vs 真实标签准确率：`0.537143`
- 定点 vs 真实标签准确率：`0.545714`
- RTL vs 真实标签准确率：`0.545714`

### 讲解话术

从最终结果来看，最重要的结论是 RTL 与 Python 定点结果完全一致，也就是一致率达到了 `1.000000`。这说明 RTL 实现已经准确复现了 Python 定点参考链路。同时，RTL 与浮点的一致率是 `0.957143`，这意味着当前剩余差异主要来自浮点到定点的量化误差，而不是 RTL 逻辑实现错误。从真实标签角度看，定点和 RTL 的总体准确率都是 `0.545714`。

## 第 10 页 混淆矩阵与每类准确率

### 页面标题

`混淆矩阵与各类别表现`

### 页面要点

- 类别 1、2、5 的识别结果最好，准确率均为 `1.000000`
- 类别 0 的 RTL 准确率为 `0.600000`
- 类别 3、4、6 仍然是当前主要薄弱类别

### 讲解话术

从每类准确率来看，类别 1、2、5 的识别效果非常稳定，已经达到 100%。类别 0 也有一定表现，准确率是 `0.600000`。但类别 3、4、6 的准确率仍然偏低，是当前最主要的混淆来源。这里非常关键的一点是，RTL 和 fixed 的结果完全一致，所以这些弱类问题本质上不是 RTL 设计错误，而是模型本身和量化策略在这些类别上的区分能力还不够强。

## 第 11 页 浮点与定点逐层误差分析

### 页面标题

`浮点与定点逐层误差分析`

### 页面要点

- 前端卷积层总体误差较小，余弦相似度普遍较高
- 随网络逐层加深，误差会逐渐累积
- `dense2_logits` 平均绝对误差为 `0.180169`
- 尽管存在量化误差，但 RTL 仍与 fixed 完全一致

### 讲解话术

从逐层分析结果可以看到，前面几层卷积和池化的误差整体比较小，和浮点结果的余弦相似度也非常高。随着网络层数加深，量化误差会逐渐累积，特别是在后端分类层会体现得更明显，比如 `dense2_logits` 的平均绝对误差是 `0.180169`。但是这些误差是浮点到定点转换带来的，不是 RTL 的额外误差，因为 RTL 已经和 fixed 完全对齐。
